# OpenCode SSE Streaming Fix

## Problem Identified

The issue was that `opencode run "command"` was **buffering output internally** before sending it to stdout/stderr. This caused:

- SSE heartbeats worked (generated by your own server)
- No real-time output during execution
- Output only appeared after stopping execution and refreshing the page

## Root Cause

In `lib/server/cli-executor.ts` line 349:

```typescript
const child = spawn(fullCommand, {
  cwd: workingDir,
  env,
  shell: true,          // ← PROBLEM 1: Shell adds buffering
  detached: true,       // ← PROBLEM 2: Detached can lose output sync
  stdio: ['ignore', 'pipe', 'pipe'],
})
```

**Issues:**
1. `shell: true` - Shell wraps the command and adds its own buffering layer
2. No environment variables to force unbuffered output
3. Missing `--print-logs` flag for opencode CLI

### 3. Fixed the Buffering Issue (MAIN FIX)
**File:** `lib/server/cli-executor.ts`

**Changes:**
```typescript
// FORCE UNBUFFERED OUTPUT - This is the key fix!
const env = {
  ...process.env,
  ...options.env,
  // Force Node.js to flush output immediately
  NODE_OPTIONS: "--no-warnings",
  // Disable all buffering
  FORCE_COLOR: "1",
  TERM: "xterm-256color",
  // Python unbuffered (if opencode uses Python internally)
  PYTHONUNBUFFERED: "1",
  // Node.js unbuffered
  NODE_NO_WARNINGS: "1",
}

// Spawn WITHOUT shell to avoid shell buffering
const child = spawn(opencodePath, [
  "run",
  commandString,
  "--print-logs", // ← Force log output to stderr for real-time streaming
], {
  cwd: workingDir,
  env,
  shell: false,      // ← CRITICAL: Don't use shell - it adds buffering
  detached: false,   // ← Keep attached for better output tracking
  stdio: ['ignore', 'pipe', 'pipe'],
})
```

**Key Fixes:**
- ✅ Removed `shell: true` - Eliminates shell buffering layer
- ✅ Added `--print-logs` flag - Forces opencode to stream logs
- ✅ Added environment variables for unbuffered output
- ✅ Changed `detached: false` - Better output synchronization
- ✅ Pass args as array instead of string - Avoids shell parsing

## How It Works Now

1. **User starts execution** → SSE connection established
2. **Backend spawns opencode** with unbuffered output flags
3. **opencode outputs data** → Immediately flushed to stdout/stderr
4. **Backend receives data** → Calls `appendOutput()` → Calls `publishOutput()`
5. **SSE subscribers** receive the output event immediately
6. **Frontend displays** output in real-time
